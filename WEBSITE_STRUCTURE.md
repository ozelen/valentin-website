## Design Considerations

1. **Professional & Clean**: Academic aesthetic, not flashy
   - Reference: [ashleycraig.com](https://www.ashleycraig.com/) style
   - Minimal, readable typography
   - Plenty of white space

2. **Easy Navigation**: Clear menu structure
   - Top navigation bar (desktop)
   - Hamburger menu (mobile)
   - Breadcrumbs for deep pages
   - Footer with quick links

3. **Publication-Focused**: Publications should be easily browsable and searchable
   - Prominent placement
   - Clear hierarchy
   - Easy filtering

4. **Mobile Responsive**: Must work well on all devices
   - Mobile-first approach
   - Touch-friendly navigation
   - Readable on small screens

5. **Fast Loading**: Optimize for performance
   - Static site generation (Astro)
   - Optimized images
   - Lazy loading

6. **Accessibility**: WCAG compliant
   - Semantic HTML
   - Alt text for images
   - Keyboard navigation
   - Screen reader friendly

7. **SEO Optimized**: For academic search engines
   - Proper meta tags
   - Structured data (Schema.org)
   - Clean URLs
   - Sitemap

## Content Organization Tips

- **Citation Format:** Use consistent citation format (APA, Chicago, or field standard)
- **DOIs and Links:** Include DOIs and links to publisher pages, PDFs where available
- **Chronological:** Group publications by year (newest first)
- **Filtering:** Use tags/categories for filtering (year, topic, journal ranking)
- **Featured Publications:** Consider a "Featured Publications" section on homepage
- **Impact Metrics:** Include citation counts and journal rankings where appropriate
- **Avoid CV Duplication:** Link to full CV instead of duplicating all content
- **Write for International Audience:** Clear, accessible language
- **Keep Pages Scannable:** Use headings, bullet points, short paragraphs
- **Easy to Maintain:** Use static files (Markdown, JSON) for easy updates

## Technical Recommendations

- **Static Site Generator:** Astro (already in use) - perfect for academic sites
- **Content Management:**
  - Publications: JSON or Markdown files
  - Pages: Astro components or Markdown
  - Easy to version control and update
  
- **Search:**
  - Simple client-side filtering (for small datasets)
  - Or Algolia for advanced search
  - Or Pagefind for static search
  
- **Features:**
  - RSS feed for publications
  - Export to BibTeX functionality
  - Print-friendly CV page
  - Sitemap.xml for SEO
  
- **Performance:**
  - Image optimization (WebP, lazy loading)
  - Minimal JavaScript
  - Fast page loads
